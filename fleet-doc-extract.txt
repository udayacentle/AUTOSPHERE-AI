Fleet Management System – Role &amp; Permission Document 1. Introduction This document defines the roles and permissions within the Fleet Management System. It ensures secure access control, operational efficiency, and proper responsibility separation using a Role-Based Access Control (RBAC) model. 2. User Roles Driver Passenger Entity Admin Super Admin Guest User 3. Role Definitions 3.1 Driver Responsibilities: View assigned trips Accept/reject trips Update trip status (Start, In Progress, Completed) Access navigation Restrictions: No access to management or reports 3.2 Passenger Responsibilities: Book or schedule rides Track trips View trip history Access basic billing Restrictions: Limited to own data 3.3 Entity Admin Responsibilities: Manage vehicles, drivers, passengers Assign and monitor trips Access reports and analytics View billing Restrictions: Limited to one organization 3.4 Super Admin Responsibilities: Full system control Manage all organizations Configure system settings Access all reports Manage roles and permissions 3.5 Guest User Responsibilities: View limited public information Restrictions: No booking or management access 4.
